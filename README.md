# Шпаргалка по Git


## Основные команды


**_1. Инициализация и клонирование_**
```
git init                        # создать новый репозиторий
git clone <url>                 # клонировать репозиторий
Статус
git status                      # статус репозитория
git log                         # история коммитов
```
---

**_2. Добавление и коммиты_**
```
git add <file>                  # добавить файл в индекс
git add .                       # добавить все файлы
git commit -m "сообщение"       # закоммитить изменения
```

---

**_3. Работа с удаленным репозиторием_**
```
git remote -v                   # список удаленных репозиториев
git push origin <branch>        # отправить изменения
```
---

**_4. Типовая последовательность работы_**
```
git status                      # проверить изменения
git add .                       # добавить файлы
git commit -m "fix: исправление бага"  # закоммитить
git pull origin main            # забрать изменения
git push origin main            # отправить изменения
```
---

**_5. Конфликты слияния_**
```
git status                      # посмотреть конфликтующие файлы
git add .                       # добавляем исправленные файлы
git commit -m "resolve conflicts" # коммитим разрешение
```
---

**_6. Удаление файлов_**
```
git rm <file>                   # удалить файл и добавить в индекс
```
---

**_7. Хеш (SHA-1)_**

Каждый объект в Git (коммит, файл, дерево) получает уникальный SHA-1 хеш — это 40-значная строка, выступающая идентификатором.​

Хеш коммита гарантирует целостность истории и позволяет точно ссылаться на изменения в репозитории.​
```
commit a1b2c3d4e5f67890123456789abcdef123456789
```
___

**_8. Лог коммитов_**
```
git log                         # полная история
git log --oneline               # краткий вид
```
